## 2. UI state / 表示モデル

### 新規 `ui/state/MemoImageUiModel.kt`

```kotlin
data class MemoImageUiModel(
    val id: String,
    val fileName: String,
    val filePath: String,
    val isPersisted: Boolean
) {
    companion object {
        fun fromDomain(
            image: MemoImage,
            resolveImagePath: (MemoImageFileName) -> String,
            isPersisted: Boolean
        ): MemoImageUiModel = MemoImageUiModel(
            id = image.id.value,
            fileName = image.fileName.value,
            filePath = resolveImagePath(image.fileName),
            isPersisted = isPersisted
        )
    }
}
```

`isPersisted` は「この画像が一度でも DB の `memo_images` に保存されたか」を表す UI 層の管理フラグ。

- 既存メモ読み込み・SavedStateHandle で保存済みと分かる画像: `true`
- Photo Picker からコピーした直後で、まだ autosave 成功前の画像: `false`
- autosave / finishEditing の `persist()` 成功後: 現在 UI state に残っている画像を `true` に更新する

このフラグは、添付直後・autosave 前に削除された画像を `RoomMemoRepository.saveMemo` の差分削除では拾えないために必要。
未保存画像を削除した場合は ViewModel が `DeleteMemoImagesUseCase` で即座にファイルを削除する。

### `ui/viewmodel/MemoEditViewModel.kt`

コンストラクタに3つ追加(既に `@Suppress("LongParameterList")` 付き):
`attachMemoImageUseCase: AttachMemoImageUseCase` / `deleteMemoImagesUseCase: DeleteMemoImagesUseCase` / `resolveMemoImagePathUseCase: ResolveMemoImagePathUseCase`

#### 添付: `fun attachImages(sourceUris: List<String>)`

```kotlin
fun attachImages(sourceUris: List<String>) {
    if (sourceUris.isEmpty()) return
    if (isFinishing || _uiState.value.isDeletePending) return
    viewModelScope.launch {
        val attached = mutableListOf<MemoImageUiModel>()
        var hasFailure = false
        sourceUris.forEach { uri ->
            try {
                val image = attachMemoImageUseCase(ImageSourceReference(uri))
                attached += MemoImageUiModel.fromDomain(
                    image,
                    resolveMemoImagePathUseCase::invoke,
                    isPersisted = false
                )
            } catch (e: CancellationException) {
                throw e
            } catch (_: Throwable) {
                hasFailure = true
            }
        }
        if (isFinishing || _uiState.value.isDeletePending) {
            // コピー中に画面が終了していたら、コピー済みファイルを即座に後始末して破棄する。
            deleteCopiedImagesQuietly(attached)
            return@launch
        }
        if (attached.isNotEmpty()) {
            updateEditState { state ->
                state.copy(images = state.images + attached, isModified = true)
            }
        }
        if (hasFailure) {
            _operationErrorEvent.trySend(MemoEditOperationErrorEvent.ImageAttachFailed)
        }
    }
}
```

`deleteCopiedImagesQuietly` は `deleteMemoImagesUseCase(attached.map { MemoImageFileName(it.fileName) })` を
try-catch(CancellationException 再throw)で包む private suspend 関数。
`updateEditState` を経由することで既存の SavedStateHandle 下書き保存とデバウンス自動保存が発火する。

#### 削除(1枚): `fun removeImage(imageId: String)`

```kotlin
fun removeImage(imageId: String) {
    if (isFinishing || _uiState.value.isDeletePending) return
    val removed = _uiState.value.images.firstOrNull { it.id == imageId } ?: return
    updateEditState { state ->
        state.copy(images = state.images.filterNot { it.id == imageId }, isModified = true)
    }
    if (!removed.isPersisted) {
        viewModelScope.launch {
            deleteCopiedImagesQuietly(listOf(removed))
        }
    }
}
```

DB に保存済みの画像はここではファイル削除しない(次回 `persist()` 時に `RoomMemoRepository.saveMemo` が差分検出して削除する。
自動保存が失敗しても参照切れにならないための設計 — 00 の決定4)。
未保存画像は DB に存在せず Repository が差分検出できないため、この場で `DeleteMemoImagesUseCase` により即時削除する。

#### 既存メソッドの変更

- `persist()` の `SaveMemoCommand` 構築に追加:
  `images = state.images.map { MemoImage(MemoImageId(it.id), MemoImageFileName(it.fileName)) },`
- `isContentBlank()` を変更: `title.isBlank() && body.isBlank() && images.isEmpty()`
  (**必ず `SaveMemoUseCase` 側の緩和(ブランチ1で実施済み)とセットであること** — 00 の gotcha 4)
- `Memo.toUiState()` に追加:
  `images = memo.images.map { MemoImageUiModel.fromDomain(it, resolveMemoImagePathUseCase::invoke, isPersisted = true) },`
- `applySavedMemo(memo)` では、現在 UI state に残っている画像を `isPersisted = true` に更新し、更新後の state を `persistSavedState` にも反映する。
  これを忘れると、autosave 後の画像を削除したときに保存済み画像まで即時ファイル削除して参照切れを起こす。
- SavedStateHandle 下書き(既存の title/body/tagIds/isFavorite と同様に):
  - companion object にキー追加: `EDIT_IMAGE_IDS_KEY = "editImageIds"` / `EDIT_IMAGE_FILE_NAMES_KEY = "editImageFileNames"` /
    `EDIT_IMAGE_PERSISTED_FLAGS_KEY = "editImagePersistedFlags"`
  - `persistSavedState`: `savedStateHandle[EDIT_IMAGE_IDS_KEY] = ArrayList(state.images.map { it.id })` と
    `ArrayList(state.images.map { it.fileName })` と `ArrayList(state.images.map { it.isPersisted })`(index 対応の3本)
  - `savedStateEdit`: 3本を読み、index で対応付けて
    `MemoImageUiModel(id, fileName, filePath = resolveMemoImagePathUseCase(MemoImageFileName(fileName)), isPersisted = flag)` を復元
    既存下書きとの互換のため flags が不足している場合は `false` と扱う
  - `clearSavedState`: 3キーを remove
- `MemoEditOperationErrorEvent`(同ファイル末尾の sealed interface)に `data object ImageAttachFailed` を追加

## 7. 検証コマンドと Done 条件

```sh
./gradlew app:ktlintCheck app:detekt :app:lintProdDebug :app:testProdDebugUnitTest
./gradlew :app:connectedDevDebugAndroidTest
```

Done 条件:

- 上記すべて green(lint は strings 片言語漏れで落ちるので注意)
- import 確認文言が ja/en ともに「画像は export/import 対象外」「同一ID上書きで添付画像が失われる」を説明している
- 手動確認(devDebug、開発端末は事前にアンインストール→インストール):
  1. 新規メモ → 画像を複数枚添付 → 戻る → 一覧カードに先頭画像のサムネイルが出る
  2. 再編集で画像が残っている(順序どおり)
  3. 画像を1枚削除 → 1秒待つ(自動保存)→ 再編集で消えている
  4. 添付直後・autosave 前に画像を削除 → `adb shell run-as com.appvoyager.litememo.dev ls files/memo_images` で未保存画像ファイルが残っていない
  5. 画像のみ(タイトル・本文空)のメモが破棄されず保存される
  6. メモをゴミ箱 → 完全削除 → `adb shell run-as com.appvoyager.litememo.dev ls files/memo_images` でファイルが消えている
  7. 端末の言語を英語にして文言・レイアウト崩れがないこと(ライト/ダーク両テーマ)
